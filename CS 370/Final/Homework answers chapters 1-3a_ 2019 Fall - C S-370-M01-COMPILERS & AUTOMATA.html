    Homework answers chapters 1-3a

    These are my own answers to most of the suggested homework problems. Some that are "!" or "!!" are not done just due to my own time constraints. This content is mostly unformatted, pasted from a text file.

    p51, Exercises for 2.2: all (2.2.1-2.2.6); this exam won't have proofs but 2.2.5 is good thought practice anyways.

    2.2.1
    a) "aa+a*" : S -> S S * -> S a * -> S S + a * -> S a + a * -> a a + a *
    b) connect the stages above with the proper edges.
    c) a postfix binary operator language with leftmost precedence; operators are + and * and a is a value.

    2.2.2
    a) 0^n1^n, where n >= 1
    b) prefix binary operator, like 2.2.1.c but with prefix operators
    c) sequences of properly nested parentheses (multiple nestings)
    d) any (pretty sure) string with equal a's and b's
    e) algebraic expression notation (multiply is "no operator" as in algebra; * is unary postfix)?

    2.2.3
    a) not ambiguous
    b) ambiguous
    c) ambiguous
    d) ambiguous
    e) ambiguous

    2.2.4
    a) (for +,* ops): E -> E + T | T : T -> T * F | F : F -> number | ( E )
    b) L -> id | L , id
    c) L -> id | id , L
    d) similar to (a) but with more operators
    e) did not do

    2.2.5
    a) base cases: 11 and 1001 are both divisible by 3, quotients are 1 and 3;
    induction 1: assume "num" is divisible by 3, then "num 0" is divisible by 3 since the value is *2, quotient will just be *2
    induction 2: (naming rules as "num2 num1") assume num1 is divisible by 3 with quotient q1; assume num2 is divisible by 3 with quotient q2; num2 in the resulting rule is N bits higher, where N=|num1|; num2 in its position is equivalent to N applications of
    induction rule 1, so value of num2 in position is num2*2^N, and its quotient is q2*2^N. Thus the entire "num2 num1"
    is divisible by 3 with quotient q2*2^N + q1. Example: "111001" -> "11 1001", quotient = 1*2^4 + 3 = 19; value of 111001 is 32+16+8+1 = 57, 57/3 = 19
    b) I think yes (checked up to 18)

    2.2.6
    roman numeral rules:
    - values must be descending right to left
    - but can have one subtraction in front
    - and must limit digit to less than previous
    - easiest is to explicitly encode multiple digits into rules
    - the below is close, I think
    mnum -> M mnum | M dnum | CM lnum | DM cnum | dnum
    dnum -> D cnum | CD lnum | cnum
    cnum -> C lnum | CC lnum | CCC lnum | lnum
    lnum -> L xnum | XL vnum | xnum
    xnum -> X vnum | XX vnum | XXX vnum | vnum
    vnum -> V inum | IV | inum
    inum -> I | II | III | empty

    p68, Exercises for 2.4: 2.4.1.a (b and c cover ideas we did not do in this chapter).

    2.4.1

    a)
    function nontermS() {
      switch (lookahead) {
       case '+': match('+'); nontermS(); nontermS(); break;
       case '-': match('-'); nontermS(); nontermS(); break;
       case 'a': match('a'); break;
       default: print_error();
      }
    }
    b)
    function nontermS() { nontermR(); } // apply transform p67-68
    function nontermR() {
      switch (lookahead) {
       case '(': match('('); nontermS(); match(')'); nontermR(); break;
       default: break;
      }
    }
    c)
    function nontermS() {
      match('0'); // KEY: both rules need this right away
      switch (lookahead) {
       case '0':
          nontermS(); // KEY: let recursive call match the 0
          match('1');
          break;
       case '1': match('1'); break;
       default: print_error();
      }
    }

    p114, Exercises for 3.1: 3.1.1

    3.3.1
    - each keyword should be its own lexeme, and its own token type (maybe "float" should be a token type "builtin-datatype")
    - each operator should be its own lexeme, its token type is its class (e.g., "<=" is in the relational-operator class)
    - the comment should be skipped, or should be one whole lexeme of type "comment" (really, it should just be skipped)
    - braces and parens and semicolons are all unique lexemes and unique types - "limitedSquare" and "x" are lexemes of type "id"

    p125, Exercises for 3.3: 3.3.2-3.3.6

    3.3.2
    a) strings of a's and b's that begin and end with a.
    b) any string of a's and b's (and the empty string).
    c) any string of a's and b's that ends with either aaa, aba, aab, or abb.
    d) any string of a's and b's that has exactly 3 b's in it.
    e) hmmm, all even-length strings of a's and b's?

    3.3.3
    a) N+1 (including empty string)
    b) same as (a)
    c) N-1 (not itself and not empty)
    d) Examples: abc -> empty,a,b,c,ab,bc,abc = 7
    abcd -> empty,a,b,c,d,ab,bc,cd,abc,bcd,abcd = 11
    works out to: (N*(N+1)/2)+1 (+1 is the empty string)
    e) Examples: abc -> empty,a,b,c,ab,bc,ac,abc = 8
    abcd -> empty,a,b,c,d,ab,bc,cd,ac,ad,bd,abc,bcd,abd,acd,abcd = 16
    works out to: sum(choose(N,k)) 0<=k<=N, which is 2^N (well known result)

    3.3.5
    a) [a-z]*a[a-z]*e[a-z]*i[a-z]*o[a-z]*u[a-z]* (doesn't exclude)
    b) a*b*c*d*e*...
    c) /s(([^s]*s[^/][^s]*)|(Q))*s/
    - using 's' to represent '\*' for readability, and Q for "*/" - the first inner part refuses any */ sequence, and then Q allows for it inside quotes; the whole thing is *'d so it can repeat
    d) Default answer: just enumerate single-each-digit string and OR them, but this isn't satisfying; hard problem!
    e)
    f) use two copies of 3.3.2.e and require a b in between them
    g) if you are interested in chess, go for it...
    h) ...
    i) (b*aa*ba*)|(b*a*) - must allow only one b after at least one a

    3.3.6
    a) [a-jA-J]
    b) [b-df-hj-np-tv-z]
    c) [0-9a-fA-F]
    d) [\.\?!]
