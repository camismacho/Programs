    Final Exam

    • Due Dec 10 by 1pm
    • Points 100

    The description below may be edited up to Thanksgiving day, but it is mostly complete.

    The final exam is Tuesday, Dec 10, 2019, 1:00pm -- 3:00pm. Not turning in a final exam will result in failing the course.

    The final exam will be a cumulative exam, but of course it will have significant content from bottom-up parsing, textbook sections 4.5 and 4.6. Topics from these sections are: shift-reduce parsing; handles; shift-reduce and reduce-reduce conflicts; LR parsing; items; LR(0) automaton; closure of item sets; goto between item sets; constructing LR parsing table from LR(0) automaton and item sets; parsing using the LR parsing tables. There will be a question asking you to use the table-driven LR parsing algorithm (Fig 4.36,p251) to parse a string, and the algorithm will not be given on the exam (the parsing table will be). Other definitions will be given, as described below.

    It will also have questions that are based on the compiler programming assignments. These can include topics such as: code generation for specific language constructs; definition and usage of the abstract syntax tree intermediate representation; definition and use of the symbol table; 'lex' usage; 'yacc' usage; action blocks for yacc grammar rules; implementation of local variables and arguments/parameters; and other topics related to our compiler project.

    Prior important topics are: regular expressions and finite automata; difference between NFA and DFA; context free grammars and context free languages; the language hierarchy; top-down recursive descent parsing; top-down table driven LL parsing; left recursion; sentential forms; leftmost and rightmost derivations;

    Good study questions from sections 4.5 and 4.6

    4.5.1 (all)

    4.5.2 (all)

    4.5.3 (all)

    4.6.2 ("augmented" means with the added S'->S rule)

    4.6.3

    4.6.4 (also should create your own sample string in the language, and show parsing of it)

    Definitions given to you on exam:

    Figure 4.32 (p245), algorithm for computing Closure(I), where I is an item set.

    Definition of the GoTo(I,X) function (top of p246), where I is an item set and X is a grammar symbol.

    Figure 4.33 (p246), algorithm for computing all canonical LR(0) item sets.

    FIRST(α) = the set of terminals that begin all strings derivable from α, where α is a sequence of grammar symbols (terminals and nonterminals). Can also include (empty).

    FOLLOW(A) = the set of terminals that can immediately follow the nonterminal A in any sentential form. We include $ as an artificial end-of-string terminal.

    Requirements for a grammar to be LL(1):
    1) FIRST(α) is disjoint for all production rules of a nonterminal S;
    2) if one production rule for S produces (empty) , then FIRST()’s of all S rules are disjoint from FOLLOW(S)
