    Program 3: Test Coverage

    In this assignment you will get experience using Ant to automate some testing, plus you will get exposure to a tool that shows you what portion of your program actually executed during your tests. Cool stuff!

    In the end, you will submit your final Ant build file (build.xml) that has your whole test suite in it.

    Procedures

        1. Unzip coverage1.zip into your programs folder (that is linked to your GitHub repo).
        2. Download and install the JaCoCo tool somewhere. Perhaps you should make a "tools" folder in your home folder, or something like that. It's up to you, but look down at the submission instructions – you might want to create this in your CS371 repository workspace. Note that the complete path should NOT contain spaces (otherwise JaCoCo will break).
        3. Edit the build.xml file and make the path in the property block (line 6) point to your installation of JaCoCo.
        4. Run 
          ant compile
          This tells ant to run the target named "compile". It should compile the two Java files and produce the class files. Try running one of them, e.g., 
          java IglooScore1 1 2 3
          It should run. Read their comments to see what they do (they are supposed to do the same thing, but are implemented differently).
        5. Run 
          ant clean
          This should delete the class files. (It deletes other stuff too, but you don't have this yet.)
        6. Run 
          ant
          This will run whatever the default target is in the build.xml file, which for this one is "jacocoreport". But since this target depends on running the program(s) under jacoco, and the running depends on the compiling, a whole bunch of things should happen! The great thing with build files is that once you create the build file, all this happens automagically! If you have JaCoCo installed correctly and your build file pointing to it, this should result in a "BUILD SUCCESSFUL" message and should result in a "report" folder now existing in your directory. If you don't have this, something went wrong, so you'll have to debug your setup.
        7. When step 6 is successful, you should have seen this in the output: 
          jacocorun:
               [echo] TEST IglooScore1: 8 9 6
               [java] Overall score: 17
               [echo] TEST IglooScore2: 8 9 6
               [java] Overall score: 17
          This is produced from the "jacocorun" target, and represents executing two Java programs, first IglooScore1 with the designated parameters, and then IglooScore2 with the same parameters. The outputs of both are correct (the sum of the two highest scores).
        8. Point a web browser to the "index.html" file in the "report" folder. You'll see some coverage statistics, and you can click around and explore them, even down to a source-code view with lines of code highlighted green, yellow, or red (non-colored lines are not executable lines). Green means that yes, this line was executed at least once in your test suite (the tests defined in the build file); yellow means the line was partially executed (mouse over and look at the pop-up message); read means the line was not executed at all.
        9. Now, your job is to create a complete test suite that achieves 100% coverage of both of these programs. For each test you must create the proper "echo" message and the corresponding Java execution. And, you need to document, and then fix, any functional errors in the programs for valid inputs. If you find functional errors for invalid inputs, document them but don't worry about fixing them.
        10. Commit and push all your changes, then tag the commit with 
          program3
